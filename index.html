JavaScript concepts

JavaScript Async

JavaScript async functions are part of its asynchronous programming model, allowing you to write asynchronous code in a way that looks synchronous, making it more readable and easier to work with. JavaScript is single-threaded, meaning it executes one thing at a time. However, tasks like fetching data from a server or reading a file take time, and instead of waiting and blocking other operations, JavaScript uses asynchronous patterns to handle these tasks.

The async keyword is used to declare an asynchronous function. It ensures that the function always returns a promise, and wraps its return value in a Promise.resolve(). If the function throws an error, it will automatically return it as a rejected promise.


        async function fetchData() {
            return "Data fetched!";
          }
        fetchData().then(data => console.log(data));  // "Data fetched!"

JavaScript Callback

JavaScript functions are first-class objects, meaning they can be passed as arguments, returned by other functions, and assigned to variables. A callback is simply a function that you pass to another function so that it can be called after some task is done.

In this case, the callback is executed immediately, as part of a normal, synchronous operation:


        function greet(name, callback) {
            console.log("Hello " + name);
            callback();
            }
                                
        function sayGoodbye() {
            console.log("Goodbye!");
            }
                                
        greet("Alice", sayGoodbye);

Out put


            Hello Alice
            Goodbye!

JavaScript Promise

In JavaScript, a Promise is an object representing the eventual completion or failure of an asynchronous operation. Promises provide a more structured and readable way to handle asynchronous actions compared to traditional callback functions, allowing you to chain multiple actions in a clean and manageable way.


        const myPromise = new Promise((resolve, reject) => {
        // Simulate an asynchronous operation (e.g., fetching data)
        let success = true;
                            
        if (success) {
            resolve("Operation successful!");
        } else {
            reject("Operation failed.");
            }
        });

JavaScript Async/Await

The await keyword can only be used inside an async function. It makes JavaScript wait for a promise to resolve before proceeding with the execution of the next lines of code. Essentially, it pauses the function until the awaited promise is resolved or rejected.


        async function fetchData() {
            let response = await fetch('https://api.example.com/data');
            let data = await response.json();
            console.log(data);
        }

JavaScript Event Loop

The Event Loop in JavaScript is a fundamental concept that manages how asynchronous code is executed and ensures that operations like I/O (input/output) don't block the main thread. JavaScript is single-threaded, meaning it can only execute one task at a time, yet it handles multiple tasks seemingly in parallel thanks to the event loop.

How the Event Loop Works In JavaScript, especially in the browser or Node.js, the execution of code happens through a combination of:

  1. Call Stack
  2. Task Queue (Callback Queue)
  3. Microtask Queue
  4. Event Loop
